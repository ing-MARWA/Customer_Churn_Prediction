## Overview

ChurnBot is a full-stack web application designed for telecom marketing teams. Instead of filling out a form, users simply chat with the bot — it collects 19 customer attributes through natural conversation, runs them through a trained **Random Forest classifier**, and returns:

- A clear **WILL CHURN / WILL NOT CHURN** verdict
- A **churn probability percentage**
- A **plain-English business explanation** (4–6 bullet points) generated by Gemini
- A **full customer profile summary** for reference

---

## How It Works

```
User types in chat
       ↓
FastAPI backend → Gemini (gemini-2.5-flash) collects 19 fields conversationally
       ↓
Once all fields are collected, Gemini outputs a structured JSON block
       ↓
Random Forest model runs prediction on the encoded features
       ↓
Gemini generates a natural-language explanation of the result
       ↓
Result card displayed: verdict + probability + explanation + profile table
```

## Installation

**1. Clone the repository**

```bash
git clone https://github.com/your-username/churnbot.git
cd churnbot
```

**2. Create and activate a virtual environment**

```bash
# Create
python -m venv venv

# Activate — Windows
venv\Scripts\activate

# Activate — Mac / Linux
source venv/bin/activate
```

You should see `(venv)` at the start of your terminal prompt.

**3. Install dependencies**

```bash
pip install -r requirements.txt
```

## Configuration

**Getting a Gemini API Key**

1. Go to [Google AI Studio](https://aistudio.google.com)
2. Sign in with your Google account
3. Click **Get API key** → **Create API key** and copy it

**Adding your key**

Open (or create) `config.json` in the project root:

```json
{
  "GEMINI_API_KEY": "YOUR_API_KEY_HERE",
  "GEMINI_MODEL": "gemini-2.5-flash"
}
```

Replace `YOUR_API_KEY_HERE` with your actual key and save the file.

> 🔒 **Keep your key safe.** Add `config.json` to `.gitignore` and never share it publicly. Your API key is linked to your Google account and may have usage or billing implications.

## Running the App

Once the virtual environment is active and `config.json` is configured:

```bash
uvicorn main:app --reload --port 8000
```

You should see:

```
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
✅  ML model & encoders loaded.
INFO:     Application startup complete.
```

Open your browser and navigate to: **http://127.0.0.1:8000**

To stop the server, press `CTRL + C` in the terminal.

> If port 8000 is taken, use `--port 8001` and access `http://127.0.0.1:8001`

## Using ChurnBot

**Starting a conversation**

When the interface loads, ChurnBot greets you and begins collecting customer information — one question at a time, conversationally.

**Fields collected (19 total)**

| Field | Valid Values |
|---|---|
| Gender | Male / Female |
| Senior Citizen | Yes / No |
| Is Married | Yes / No |
| Dependents | Yes / No |
| Tenure | Number of months (e.g. `24`) |
| Phone Service | Yes / No |
| Dual (multi-line) | Yes / No / No phone service |
| Internet Service | DSL / Fiber optic / No |
| Online Security | Yes / No / No internet service |
| Online Backup | Yes / No / No internet service |
| Device Protection | Yes / No / No internet service |
| Tech Support | Yes / No / No internet service |
| Streaming TV | Yes / No / No internet service |
| Streaming Movies | Yes / No / No internet service |
| Contract Type | Month-to-month / One year / Two year |
| Paperless Billing | Yes / No |
| Payment Method | Electronic check / Mailed check / Bank transfer (automatic) / Credit card (automatic) |
| Monthly Charges | Dollar amount (e.g. `65.50`) |
| Total Charges | Dollar amount (e.g. `1570.00`) |

**Reading the result**

Once all fields are collected, ChurnBot automatically runs the prediction and displays a result card with:

- **Prediction Verdict** — `WILL CHURN` or `WILL NOT CHURN`
- **Churn Probability** — percentage confidence from the model
- **Business Explanation** — 4–6 bullet points in plain language identifying key risk or retention factors
- **Customer Profile Summary** — a table of all collected values

**Starting a new prediction**

Click the **New Conversation** button in the left sidebar to reset and analyse a different customer.

## ML Model & Training

The model was built in `Customer_Churn_Prediction_using_ML.ipynb` using the [Telco Customer Churn dataset](https://www.kaggle.com/datasets/blastchar/telco-customer-churn).

**Pipeline summary:**

1. **Data cleaning** — dropped `CustomerID`, converted `Total_Charges` to float, handled 11 missing values
2. **EDA** — identified class imbalance in the `Churn` column; found high correlation between `tenure` and `Total_Charges`
3. **Preprocessing** — `LabelEncoder` applied to all categorical columns; encoders saved to `encoders.pkl`
4. **Imbalance handling** — SMOTE (Synthetic Minority Oversampling Technique) applied to the training set
5. **Model comparison** — Decision Tree, Random Forest, and XGBoost evaluated with 5-fold cross-validation
6. **Final model** — **Random Forest** selected; serialised to `customer_churn_model.pkl` along with feature names

**Key findings from EDA:**
- Tenure is the strongest retention signal — longer-tenured customers churn significantly less
- Month-to-month contracts and Fiber optic internet are associated with higher churn risk
- Customers with no tech support or online security are more likely to churn

## API Reference

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/` | Serves the chat UI |
| `POST` | `/api/new-session` | Creates a new conversation session, returns `session_id` |
| `POST` | `/api/chat` | Sends a message; returns a chat reply or a full prediction result |
| `GET` | `/api/health` | Health check — returns model and LLM status |

**POST `/api/chat` — Request body**

```json
{
  "session_id": "uuid-string",
  "message": "user message text"
}
```

**Response types:**

```json
// Regular chat message
{ "type": "message", "html": "...", "raw": "..." }

// Prediction result (triggered when all 19 fields are collected)
{
  "type": "prediction",
  "churn": true,
  "prob": 78.3,
  "no_prob": 21.7,
  "explanation_html": "...",
  "profile": [{ "field": "Contract", "value": "Month-to-month" }, ...]
}
```

## Troubleshooting

| Issue | Solution |
|---|---|
| Page not loading at `http://127.0.0.1:8000` | Make sure the `uvicorn` command is still running in the terminal and check for error messages |
| `ModuleNotFoundError` when starting server | Ensure the virtual environment is activated (`(venv)` visible in prompt) and `pip install -r requirements.txt` has been run |
| `Invalid API key` or 502 error | Open `config.json` and verify the Gemini API key has no extra spaces or surrounding quotes |
| ChurnBot not responding to messages | Check your internet connection — Gemini requires an active connection. Also verify the API key is valid |
| `Address already in use` on port 8000 | Run with a different port: `uvicorn main:app --reload --port 8001` and access `http://127.0.0.1:8001` |
| `customer_churn_model.pkl not found` | Make sure you are running the `uvicorn` command from inside the project folder, not a parent directory |

## Security Notes

- **Never commit `config.json`** — add it to `.gitignore` immediately
- The app uses **BeautifulSoup4 to sanitise all LLM HTML output** before rendering it in the browser, preventing XSS from model responses
- Session data is stored **in-memory only** — all sessions are lost when the server restarts
- There is no authentication layer by default — do not expose this app on a public network without adding one
